# Stage Engine v2 视觉圣经

文档版本：v1.0  
更新日期：2026-07-18  
适用范围：沙盘编辑器、Stage Engine v2、沙具资产、天气昼夜、AI 伙伴浮层、左右工作面板

---

## 1. 视觉 Thesis

温暖、清澈、可触摸的桌面微缩沙盘：浅色木框、细腻沙面、Q 版玩具沙具、柔和日光/夜光、低噪声 UI，让用户感到安全、专注和愿意表达。

这个项目的视觉目标不是“炫酷游戏场景”，而是“可长期凝视、可表达内在感受、具备专业产品可信度的微缩心理沙盘工作台”。

---

## 2. 参考方向与禁止方向

### 2.1 参考方向

- 桌面上的真实方形木框沙盘。
- 微缩玩具模型。
- Q 版软塑料 / 粘土 / 手工木质道具。
- 正交或弱透视的斜俯视相机。
- 明亮、温和、低攻击性的心理陪伴产品。
- 类游戏的触感，但不做游戏首页和营销化英雄区。

### 2.2 禁止方向

- 把沙盘改成开放世界、浮岛、农场、塔防地图。
- 用黑色科幻 UI 包住心理沙盘。
- 用大量玻璃拟态、厚边框、强霓虹造成阅读压力。
- 将沙具做成扁平图标再加阴影冒充 3D。
- 为了装饰牺牲拖拽、选择和分析可读性。
- 继续在旧 Konva 舞台上无限叠加最终修饰 pass。

## 3. 构图原则

### 3.1 工作台层级

页面优先级必须稳定：

1. 中央沙盘是主视觉和主操作区。
2. 左侧沙具库是拿取工具区。
3. 右侧作品面板是洞察和属性区。
4. AI 伙伴是辅助陪伴，不应压住沙盘主体。
5. 顶部导航只提供模式切换和账户状态。

### 3.2 沙盘视角

Stage Engine v2 默认使用正交相机：

- 俯仰角：约 48 到 58 度。
- 水平旋转：默认轻微左偏 5 到 12 度。
- 相机必须能展示木框厚度、沙面、沙具体积和落影。
- 任何旋转和平移都不能让方形沙盘主体完全离开画面。

### 3.3 留白

- 沙盘周围需要留出桌面环境，但不能喧宾夺主。
- 面板内部信息密度要高，但不要堆满说明文字。
- 资产库卡片以清晰识别为第一目标，不能为了网格数量压缩到看不清。

## 4. 色彩系统

### 4.1 日间主色

| 用途 | 颜色建议 | 说明 |
|---|---|---|
| 沙面 | `#d9bd84`, `#ecd9ad`, `#f3e4bd` | 温暖浅沙色，有颗粒变化 |
| 木框 | `#9b6a35`, `#c9965c`, `#e0b37a` | 木纹清晰，边缘有油润高光 |
| 内衬蓝 | `#3d9fc0`, `#6fc7dd` | 明亮但不刺眼 |
| UI 背景 | `#f7efe0`, `#fff9ed` | 类纸面、暖白 |

| 危险操作 | `#c9553d` | 删除、清空 |
| 次级文字 | `#68756c` | 保持可读，不发灰 |

### 4.2 夜间主色

| 用途 | 颜色建议 | 说明 |
|---|---|---|
| 页面背景 | `#08141b`, `#10232d` | 深蓝绿，不纯黑 |
| 面板背景 | `#13252e`, `#182f38` | 与背景有层次 |
| 面板边框 | `#385863` | 不要过亮 |
| 正文文字 | `#e8f4ef` | 必须清晰 |
| 次级文字 | `#a9c1bb` | 不低于可读阈值 |

| 主强调 | `#70d7c8` | 夜间高可见 |

### 4.3 颜色约束

- 一个视图内最多一个主强调色。
- 风险标签可以有少量色彩差异，但不能成为视觉噪声。
- 夜间模式禁止低透明白字叠在浅色卡片上。
- 输入框、按钮、表格、标签必须在日夜主题中分别检查。

## 5. 材质语言

### 5.1 沙面

目标：可触摸、有颗粒、有轻微起伏，但不脏、不灰、不像噪声贴图。

必须包含：

- 微颗粒法线或程序化 bump。
- 低频沙丘起伏。
- 轻微手工刮痕。
- 沙具接触处的暗部压痕。
- 光照变化下可见的浅高光。

禁止：

- 大面积脏污。
- 过强颗粒造成阅读干扰。
- 雨天滤镜把沙具压暗到看不清。

### 5.2 木框

目标：真实木质玩具盒，有厚度、倒角、边缘高光和端面纹理。

必须包含：

- bevel。
- 木纹方向。
- 角部拼接。
- 内外侧明暗差。
- 前沿厚度。

### 5.3 沙具

统一为 Q 版玩具风：

- 圆润边缘。
- 体块明确。
- 表情温和。
- 低复杂度几何，但不是低质量。
- 明确落地阴影和接触暗部。

材质 preset：

| Preset | 用途 |
|---|---|
| `softPlastic` | 人物、动物、机器人外壳 |
| `claySkin` | 人物皮肤和温和表情 |
| `paintedWood` | 房子、围栏、桥、塔 |
| `warmCeramic` | 石头、特殊象征 |
| `toyMetal` | 机器人、工具、局部装饰 |
| `glassWater` | 水域、泡泡、光泽面 |
| `sandMatte` | 沙面、足迹、底座 |

## 6. 灯光规范

### 6.1 日间晴天

- 主方向光来自左上前方。
- 阴影清晰但柔和。
- 沙具颜色饱和，边缘有暖高光。
- 桌面背景明亮，不能过曝。

### 6.2 阴天

- 降低主光强度。
- 提高环境光。
- 阴影更软更短。
- 背景云层可见但低对比。

### 6.3 雨天

- 主体对比度略降，但沙具轮廓必须清晰。
- 雨线不能遮挡拖拽点和选中框。
- 沙面有轻微湿润高光。
- UI 不跟随雨水效果变脏。

### 6.4 夜间

- 沙盘可读性优先。
- 月光或柔和顶光提供基础照明。
- 光源沙具可以有局部发光反馈，但不能让其他沙具不可见。
- 页面 UI 同步深色主题。

## 8. UI 风格约束

### 8.1 产品界面

- 工具界面要像高级创作软件，而不是营销首页。
- 顶部标题减少长描述，用状态 chip 承载信息。
- 左侧沙具库像游戏背包，但必须保持清晰和效率。
- 右侧面板默认只展示核心洞察，其余折叠。
- 管理后台列表主导，详情用抽屉，不把所有字段堆在一屏。

### 8.2 字号

| 用途 | 建议 |
|---|---|
| 页面主标题 | 24 到 30px |
| 面板标题 | 18 到 22px |
| 正文 | 14 到 16px |
| 表格/标签 | 12 到 14px |
| 对话正文 | 15 到 17px |

禁止在对话气泡和卡片里使用过大的正文，避免像海报字。

## 9. 动效规范

动效要服务触感和反馈：

- 沙具拿起：轻微上浮和阴影增强。
- 沙具放下：短促缩放回弹，接触阴影稳定。
- 选中：细线框和控制点淡入。
- 天气切换：光照和背景渐变，不闪烁。
- AI 伙伴：呼吸、眨眼、轻微点头，不抢注意力。
- 面板抽屉：快速滑入，保持舞台主体可见。

禁止：

- 大量循环动效。
- 会干扰拖拽判断的浮动。
- 夜间模式强闪烁。

---

## 10. 验收判断

一张标准截图必须同时满足：

- 3 秒内能看出这是心理沙盘，不是普通游戏地图。
- 沙盘木框、沙面、沙具都有体积。
- 12 个 hero 沙具至少 10 个能清晰识别。
- UI 不遮挡沙盘主体。
- 日夜模式文字都能读。
- 拖拽和选择控件看起来像专业工具，而不是临时调试层。
